# 03. 크리에이터판 디자인 시스템 — "다크 레이더 관제실"

> 컨셉 한 줄: **밤의 관제실** — 어두운 방에서 스크린만 빛나는, 신호를 감시하는 공간.
> 원작과의 구분: 원작은 군사 조준(Target) 은유. 우리는 **천문 관측(Observatory)** 은유 — 스캔·신호·궤적. 용어와 형태 언어를 여기서 가른다.
> 실물 기준점: `design/creator-radar-mockup.html`

## 1. 컬러 토큰

```css
:root {
  /* 배경 계층 (깊이 3단) */
  --bg-space:    #0A0E1A;  /* 앱 배경 — 진공 */
  --bg-panel:    #101624;  /* 패널/카드 표면 */
  --bg-raised:   #17202F;  /* 호버/선택/입력 필드 */

  /* 라인 */
  --line-subtle: rgba(125,155,200,.14);  /* 기본 보더 */
  --line-focus:  rgba(77,216,255,.45);   /* 포커스 링 */

  /* 시그널 컬러 (의미 고정 — 혼용 금지) */
  --signal-naver:#00E09E;  /* 네이버 축 점수 — 그린 */
  --signal-ai:   #4DD8FF;  /* AI 축 점수 — 시안 */
  --signal-hot:  #FFB020;  /* 급상승/주의 — 앰버 */
  --signal-down: #FF5C7A;  /* 하락/에러 — 로즈 */

  /* 텍스트 (대비 검증: bg-space 기준) */
  --text-hi:     #EAF1FB;  /* 15.2:1 — 제목/점수 */
  --text-body:   #B7C3D6;  /*  9.1:1 — 본문 */
  --text-dim:    #7E8CA3;  /*  5.2:1 — 보조 (4.5:1 이상 유지) */

  /* 글로우 (스크린샷 바이럴의 핵심 — 남용 금지: 화면당 글로우 요소 3개 이하) */
  --glow-naver:  0 0 24px rgba(0,224,158,.25);
  --glow-ai:     0 0 24px rgba(77,216,255,.22);
}
```

**규칙**: 그린=네이버, 시안=AI를 앱 전체에서 절대 바꾸지 않는다(색만으로 축을 읽게). 색맹 대응: 축 표기는 항상 색+라벨(`N`/`AI`) 병기.

## 2. 타이포그래피

| 토큰 | 폰트 | 크기/행간 | 용도 |
|------|------|----------|------|
| display | Pretendard 700 | 28/36 | 화면 제목, TOP SIGNAL |
| score-xl | **JetBrains Mono 700** | 40/44, tabular | 상세 패널 대형 점수 |
| score | JetBrains Mono 600 | 18/24, tabular | 카드 점수, 티커 숫자 |
| title | Pretendard 600 | 17/24 | 카드 키워드명 |
| body | Pretendard 400 | 14/22 | 설명문 |
| caption | Pretendard 400 | 12/18 | 각주, 방법론 표기 |

숫자는 전부 모노스페이스+tabular — 관제실 느낌의 절반은 숫자 정렬에서 나온다. 본문은 Pretendard로 가독성 확보(모노 남용 금지).

## 3. 스페이싱·형태

- 스페이스 스케일: 4/8/12/16/24/32/48 (Tailwind 기본과 호환)
- 라운드: 카드 12px, 칩/뱃지 999px, 패널 16px
- 카드 보더: 1px `--line-subtle` + 배경 `--bg-panel`. 선택 시: 보더가 해당 축 시그널 컬러 45% + 글로우
- 레이더 동심원: 배경 장식은 opacity 0.04~0.08 — 콘텐츠 대비를 침범하지 않는 선

## 4. 핵심 컴포넌트

### C-01 KeywordCard
```
┌──────────────────────────────┐
│▌제주 비오는날 갈만한곳            │ ▌= 클러스터 스트라이프 3px
│▌                              │
│▌ N 87 ━━━━━━━━░░  AI 92 ━━━━━━━━━░│ DualScore: 축별 미니바+숫자
│▌ ▲ 급상승 · 문서 적음     ⌁spark │ 상태 칩 + 7일 스파크라인
└──────────────────────────────┘
```
- 크기 고정 h 132px (그리드 정렬), 제목 2줄 말줄임
- hover: bg-raised + translateY(-2px) 120ms / 선택: 시그널 보더+글로우
- AI 점수 미측정: 바 대신 `측정 대기` 칩 (시안 점선 보더) — 0점 표시 절대 금지

### C-02 DualScore (원자 컴포넌트)
`[N] 87` `[AI] 92` — 축 라벨 칩(축 컬러 배경 12% + 축 컬러 텍스트) + 모노 숫자. 크기 sm/md/xl 3종. 모든 화면에서 이 컴포넌트만 사용(점수 표기 통일).

### C-03 GaugeRing
SVG 원형 게이지 (S4 진단 점수용). 0→값 카운트업 900ms ease-out + 스트로크 채움 동기화. 중심: score-xl 숫자 + 등급 라벨.

### C-04 LiveTicker
하단 고정 32px 스트립. 콘텐츠: `키워드 · N점수 · AI점수` 반복. CSS transform 마퀴(60s 루프), hover 일시정지, 클릭 시 해당 카드로 스크롤+선택. reduced-motion: 마퀴 대신 5초 페이드 로테이션.

### C-05 RadarSweep (스캔 중 전용)
동심원 4개 + 회전 스윕 그라데이션(8s linear). 발견 키워드가 궤도 위 점으로 찍히며 카드 그리드로 "날아가는" 전이(첫 스캔의 마법 순간 — 90초를 볼거리로). GPU: transform/opacity만 사용.

### C-08 StarChart (성도 뷰) — 간판 컴포넌트
S2 기본 뷰. 옵시디언 그래프의 대체이자 반박: **배치에 의미가 있는 화려함.**
- 반경 = 점수 (중심 100 → 외곽 40). 등고선 링 3개(90/70/50)에 점수 라벨.
- 각도 = 클러스터 부채꼴. 클러스터 팔레트: 시안/그린/앰버/바이올렛(#A78BFA) 순환. 성좌선: 같은 클러스터 노드 연결, 1px, 클러스터 컬러 25%.
- 노드: 코어(6~14px, 점수比) + 글로우 헤일로(radial-gradient) + 키워드 라벨. TOP 3는 라벨+DualScore 상시, 나머지는 hover 시.
- 스윕: conic-gradient 원뿔(24° 폭) 12s/회전. 스윕 통과 시 노드 밝기 1.4× 점등 (CSS animation-delay를 각도에서 계산).
- 배경: 미세 별(twinkle, 40개 이하) + 비네트. 전부 transform/opacity — 60fps 필수.
- reduced-motion: 스윕·트윙클 정지, 정적 성도.
- **공유 프레임**: 화면 캡처 버튼 → 로고+주제+날짜 오버레이 얹은 1200×675 이미지 생성 (스레드 바이럴 규격).

**구현 오픈소스 (2026-07 리서치 확정 — 전부 MIT)**
| 역할 | 라이브러리 | 근거 |
|------|-----------|------|
| 렌더러 | **PixiJS v8 + @pixi/react** (React 19 전용 재설계) | WebGL 2D — CSS로 불가능한 진짜 글로우·블룸을 60fps로. 3D(three.js)보다 가볍고 우리 "의미 있는 2D 배치"와 맞음 |
| 광원 효과 | **pixi-filters** — 노드: `GlowFilter({distance, outerStrength})`, 씬 전체: `AdvancedBloomFilter` | 별 코어+헤일로를 필터로 — DOM 방식의 radial-gradient 한계 돌파 |
| 배치 계산 | **d3-force** — `forceRadial(d => radiusOf(d.score))`(반경=점수 유지!) + `forceCollide`(라벨 겹침 방지) + 클러스터 각도 초기값 | 스캔 완료 시 1회 프리컴퓨트(라이브 물리 아님) — 의미 유지 + 겹침 해결 |
| 공유 이미지 | Pixi `renderer.extract` | 캔버스 → PNG 네이티브 추출, html-to-image 불필요 |
| UI 모션 | motion(framer-motion) + 카운트업 | 패널 전환·점수 카운트업 |
| 랜딩 배경 | tsParticles stars preset (활발히 유지보수 중) | 랜딩 한정 — 앱 안은 Pixi 단일 렌더러 유지 |
| 딜라이트 | canvas-confetti | S5 첫 인용 발견 순간 1회 (남용 금지) |

검토 후 보류한 대안: **react-force-graph-3d + UnrealBloomPass** (공식 블룸 예제 존재, 원작을 3D로 초월 가능) — 화려함은 최고지만 3D 회전 시 "중심=고점수" 공간 의미가 무너지고 번들이 무거움. 성도가 성공해 "우주 모드" 프리미엄 이스터에그가 필요해지면 재검토.

### C-06 SignalBadge
판정 라벨 칩: `지금 쓰세요`(그린 채움) / `좋은 기회`(그린 보더) / `보통`(dim) / `관망`(dim 점선). 점수 구간은 02 문서 S3 정의 준수.

### C-07 EvidenceRow
근거 블록 행: 지표명(dim) — 값(모노, hi) — 해석문(body). S3·S4 공용.

## 5. 모션 원칙

| 대상 | 스펙 | 이유 |
|------|------|------|
| 점수 등장 | 카운트업 600~900ms, 1회만 | 숫자가 "측정되는" 감각 |
| 카드 진입 | stagger 40ms, fade+8px up | 신호가 잡히는 리듬 |
| 스윕/티커 | 상시 루프 (S2 한정) | 살아있는 관제실 |
| 전 항목 | `prefers-reduced-motion` 시 즉시 상태 전환 | 접근성 |

상시 애니메이션은 S2의 성도(스윕·트윙클)·티커로 제한. 나머지 화면은 정적 — 피로 방지.

**화려함의 원칙 (성도 뷰 도입에 따른 개정)**: 화려함은 S2 성도 뷰에 **집중 투자**하고 나머지 화면은 절제 유지. "모든 화면이 화려한 앱"은 피곤한 앱이다 — 간판 하나가 눈부시고 작업 화면은 편안해야 매일 쓴다. 글로우 요소 상한(화면당 3개)은 성도 뷰에 한해 해제.

## 5.5 반응형 — 듀얼 퍼스트 (폰에서 완결 · PC에서 최대 생산성)

**브레이크포인트**: `base`(360~) → `md`(768) 2열·패널 등장 → `lg`(1280) 3열 관제실 → `xl`(1600) 정보 밀도 확대. **CSS는 base부터 작성하고 min-width로 확장한다** (max-width 하향식 금지). 단, 이는 구현 기법이지 우선순위가 아니다 — **lg가 이 앱의 "완전체"이고 base는 "완결된 휴대판"이다. 둘 다 축소판이 아니다.**

| 요소 | Mobile (기준) | md | lg |
|------|--------------|-----|-----|
| 레이아웃 | 단일 컬럼 스택 | 성도+하단 패널 | 좌 240 / 중앙 / 우 360 3열 |
| S3 상세 | 바텀시트 85% | 바텀시트 | 우측 고정 패널 |
| 수집 채널 | 가로 칩 스크롤 | 가로 칩 | 좌 세로 패널 |
| S2.5 조회 | 풀스크린 시트 | 모달 640px | 모달 640px (⌘K) |
| 티커 | 유지 (28px) | 유지 | 유지 (32px) |
| 성도 라벨 | TOP 3 상시 | TOP 5 | TOP 8 + xl에서 전 노드 라벨 |
| 카드 그리드 | 1열 | 2열 | 3열 / xl 4열 |

**터치 규칙 (폰)**: 탭 타깃 최소 44×44 / hover에만 존재하는 정보 금지(탭=선택으로 대체) / 스와이프: 바텀시트 내리기·주제 탭 전환 / 성도는 핀치 줌·팬·더블탭 리셋. 폰트 최소 12px(caption) — 그 이하 금지.

**데스크톱 강화 규칙 (PC — 축소판 금지)**: 키보드 레이어 필수(⌘K 조회, 방향키 순회, `s` 스캔, `d` 진단, Ctrl+1~5 주제 전환) / hover 툴팁·프리뷰는 PC의 정당한 강화 수단(단, 동일 정보가 터치에서도 도달 가능해야) / 대화면은 여백이 아니라 **정보 밀도**로 보상(라벨 수·근거 펼침·열 수) / 브리핑 복사→에디터 딥링크로 PC 작업 흐름 완결.

**성도 모바일 성능 프로파일** (C-08 구현 조건): 노드 상한 40, `resolution ≤ 2`, Bloom 1단계 하향, 프레임 드랍 감지 시 Bloom 자동 오프. 검증 기기: 갤럭시 A 시리즈급(중급기)에서 60fps — 최신 플래그십 기준 금지.
