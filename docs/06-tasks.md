# 06. 태스크 분해 — 개발 실행 계획

> 표기: `[모듈] ID. 태스크 — 완료 기준` / 의존성은 `←`
> 순서 원칙(전략 §9): **A(소상공인 모듈) 먼저 → B(크리에이터판)**. A가 공유 패키지를 만들고 B가 소비.

## Phase P — 사업·법무 준비 (개발과 병행 · 리드타임 소화가 목적)

> 전부 개발이 아니라 **기다림이 긴 것들** — 시작만 먼저 해두면 개발 완료 시점에 병목이 안 된다.

- P-1. 사업자등록 확인 + **통신판매업 신고** (구독 판매 필수) ← 없음 **(사용자)**
- P-2. **토스페이먼츠 가맹 계약 + 빌링(정기결제) 심사** — 심사에 수 주 소요 가능, A3-4/B3-1의 선행 조건 ← P-1 **(사용자)**
- P-3. 약관·개인정보처리방침·**환불정책**(전자상거래법 — 구독 중도해지 기준 명시) 초안 → 두 제품 공용 ← P-1
- P-4. 카카오 비즈메시지 발신프로필 신청 (A3-5 선행 — 기존 "지금 당장" 항목을 여기로 승격) ← P-1 **(사용자)**

## Phase A — boina 레이더 모듈 + UX 재구성 (소상공인) · 예상 4~5주 (07 재설계 반영 상향)

### A0. 거버넌스 (boina 규칙 준수)
- A0-1. **04+07 문서** 기반 boina `docs/planning/` Gap Review 패치 생성 (07 §5 패치 항목 6개: 홈 피드 전환·S3+S4 통합·라우트 재편 포함) → 게이트 판정 ← 없음
- A0-2. 결제(빌링) 변경 L3 분류 확인 + DECISION_LOG 기록 ← A0-1
- A0-3. **UX 재구성 실행** — 라우트 재편(home/status/rivals/write)·홈 피드 조립·S3+S4 통합. 로직·데이터 계층 재사용, **비주얼은 08 디자인 시스템("가게 성적표")으로 전면 교체** (토큰·M-01~M-07 컴포넌트·증거 시트·변화 카드) ← A0-1

### A1. 공유 패키지 (이후 모든 것의 기반)
- A1-1. `packages/keyword-pipeline` 신설 — 검증 폴더 JS→TS 이식 (expand/시드분해/collectSignals/naverScore), 유닛테스트(니치 시드 폴백 케이스 필수) ← A0-1
- A1-2. 네이버 API 키 4종 발급 + 완전 검증 실행 (검증 폴더 체크리스트 잔여 항목) ← 없음 **(사용자 작업 선행)**
- A1-3. 쿼터 가드 모듈 (일 예산 배정·소진 시 부분 결과) ← A1-1

### A2. 데이터·잡
- A2-1. Drizzle 스키마 4테이블 (radar_subscriptions/scans/keywords/feedback) + 마이그레이션 ← A0-1
- A2-2. 주간 스캔 잡 — 가게 fan-out, 실패 격리, 재시도 1회 ← A1-1, A2-1
- A2-3. 가게명 AI 프로브 연결 (기존 geo-validator 호출) ← A2-2

### A3. 화면·알림·결제
- A3-1. 홈 카드 ④ 미구독 상태 (실측 미리보기 1건 + 블러 티저 + 구독 CTA + 예시 폴백 라벨) ← A1-1
- A3-2. 홈 카드 ④ 구독 상태 (검색어 5개 + 글감 연결 + 빈/실패 상태 + 알림톡 딥링크) ← A2-2
- A3-3. 글감 연결 — 기존 snippets 모듈에 검색어 파라미터 전달 ← A3-2
- A3-4. 토스 빌링 (빌링키 발급·해지·웹훅 서명 검증·멱등 처리·3일 실패 유예·무료 강등·환불정책 문구 일치) ← A0-2, P-3
- A3-5. 카카오 알림톡 + 문자 폴백 발송 구현 ← A2-2, P-4(발신프로필 승인)

### A4. 검증
- A4-1. E2E: 진단→홈 카드 ④ 미구독→구독→월요 잡→알림 딥링크→홈 카드 ④ 구독→글감 복붙 전체 경로 ← A3.*
- A4-2. 실가게 5곳 파일럿 2주 (열람률·"썼어요" 응답 수집) ← A4-1

## Phase B — 크리에이터판 새 레포 · 예상 5~6주

### B0. 기반
- B0-1. 엔진·파이프라인 패키지 발행 (GitHub Packages) + boina 발행 워크플로 ← A1-1
- B0-2. 레포 부트스트랩 (Next+Tailwind+Drizzle+Supabase+CI) ← 없음
- B0-3. 03 디자인 토큰 → Tailwind 설정 + 기본 컴포넌트 셸 (C-01~C-07 스토리북) ← B0-2

### B1. 레이더 코어
- B1-1. DB 스키마 전체 (05 §3) ← B0-2
- B1-2. 이중 점수 산식 `lib/scoring` — 순수 함수 + 유닛테스트 (근거 jsonb 왕복 포함) ← B1-1
- B1-3. 일일 스캔 잡 — 2단 제공(네이버 선노출·AI 후행) + SSE 스트리밍 ← B1-2, B0-1
- B1-4. S1 온보딩 (실시간 미리보기 debounce, 니치 폴백 안내) ← B0-3, B1-3
- B1-5. S2 대시보드 — 카드 그리드·필터·티커·4상태 + 성도 CSS 목업 fallback 유지 ← B1-4
- B1-5a. 성도 스파이크(반나절) — Pixi 글로우 룩·모바일 60fps·라벨 겹침·renderer.extract가 목업 CSS 대비 우위인지 실물 비교 후 채택 확정 (미달 시 CSS 버전 유지) ← B0-3, B1-5
- B1-5b. **성도 뷰(C-08) 제품 구현** — 채택된 경우에만 Pixi v8+pixi-filters(Glow/Bloom), d3-force 프리컴퓨트 배치(forceRadial=점수·forceCollide=라벨), 스윕·블립·성좌선, 60fps 검증, renderer.extract 공유 프레임 구현 ← B1-5a
- B1-6. S3 상세 패널 — 근거 블록·판정 라벨·브리핑 복사 ← B1-5
- B1-7. S2.5 즉시 조회 — ⌘K 오버레이, 단건 신호 수집 API(/api/lookups), 2단 제공(N 즉시·AI 옵트인), 한도 집행 ← B1-2, B0-3

### B2. 진단·추적
- B2-1. S4 글 진단 — runDiagnosisPipeline 연동, 체크리스트 매핑, 60s 단계 표시 ← B0-1, B1-1
- B2-2. 인용 프로브 오케스트레이션 `lib/probe` — 질문 세트 생성·버전 기록·판정 ← B2-1
