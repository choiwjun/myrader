# 02. 크리에이터판 화면 명세 (7화면)

> 디자인 토큰·컴포넌트는 `03-creator-design-system.md` 참조. 모든 화면 다크 관제실 테마.
> **반응형: 듀얼 퍼스트.** 크리에이터의 실제 사용은 기기별 역할이 다르다 —
> **PC = 작업 기지**(키워드 조사→글 진단→브리핑 복사→블로그 에디터로 이동하는 본 워크플로우, 3열 관제실),
> **폰 = 확인 기지**(알림 확인·인용 발견·이동 중 아이디어 체크).
> 수용 기준 두 개가 동급이다: ① 모든 화면이 폰에서 완결된다 ② 모든 작업이 PC에서 최대 생산성을 낸다.
> 구현 기법으로서 CSS는 모바일부터 작성(min-width 확장)하되, 설계는 어느 쪽도 "축소판"이 아니다.
> 상태 규칙: 모든 데이터 화면은 **로딩/빈/에러/부분실패** 4상태를 반드시 구현한다.

## 화면 흐름

```
S1 온보딩 ─→ S2 레이더(홈) ─→ S3 키워드 상세(패널)
                │  ├─→ S2.5 즉시 조회 (전역 검색, 모든 화면에서 진입)
                │  ├─→ S4 글 진단
                │  ├─→ S5 인용 추적
                │  └─→ S6 주간 리포트
                └─→ S7 설정/구독
비로그인: 랜딩(마케팅 페이지, 별도 정적) → 가입 → S1
```

---

## S1. 온보딩 — 주제 등록

**목적**: 90초 안에 첫 레이더가 도는 것을 보여준다. 가입 직후 빈 화면 금지.

**레이아웃**: 중앙 단일 카드 (max-w 560px), 배경에 레이더 동심원 애니메이션(저채도).
1. Step 1 — "어떤 주제를 감시할까요?" 입력 (placeholder: "예: 제주 여행, 홈카페, 사이드프로젝트")
   - 입력 시 실시간 자동완성 미리보기 5개 (파이프라인 1-hop, debounce 400ms) → "이런 키워드가 잡힙니다" 칩으로 표시. **입력만으로 가치를 먼저 보여주는 장치.**
2. Step 2(선택) — "내 채널이 있다면" 블로그/사이트 URL 입력. 스킵 가능. 입력 시 REQ-C3 추적 대상 자동 등록.
3. CTA `레이더 가동` → S2로 이동하며 첫 스캔 시작.

**상태**
- 미리보기 로딩: 칩 스켈레톤 3개.
- 미리보기 0개(니치 주제): "직접 조합해 찾아볼게요 — 지역/분야를 나눠 스캔합니다" 안내 (시드 분해 폴백 발동 예고). 진행 차단하지 않음.
- 에러: "네이버 연결이 원활하지 않아요. 가동은 예약해두고 준비되면 알려드릴게요." → 스캔 큐 등록 후 S2 진입.

**모바일**: 동일 단일 컬럼. 동심원 애니메이션 제거(성능).

## S2. 레이더 대시보드 (홈) — 앱의 얼굴

**목적**: 접속 10초 안에 "오늘 뭘 쓰면 되는지"가 보인다. 스크린샷 찍고 싶은 화면.

**레이아웃 (Desktop 3열)**
```
┌────────────────────────────────────────────────────────────┐
│ 헤더: ◉로고 | 주제 탭(제주여행 ▾) | 쿼터 게이지 | [▶ 지금 스캔] │
├──────────┬───────────────────────────────┬─────────────────┤
│ 좌 240px  │ 중앙 (유동)                     │ 우 360px         │
│ 수집 상태  │ TOP SIGNAL 스트립               │ 선택 키워드 상세  │
│ 채널 4종   │ 키워드 카드 그리드 (2~3열)        │ (S3 패널)        │
│ 신호등+    │ 카드: 키워드/이중점수/변동화살표    │ 기본: 1위 키워드  │
│ 미니차트   │ 클러스터별 좌측 컬러 스트라이프     │ 자동 선택        │
│           │                               │                 │
├──────────┴───────────────────────────────┴─────────────────┤
│ LIVE 티커: 최근 발견 키워드·점수 가로 스크롤 (8s 루프, hover 정지)│
└────────────────────────────────────────────────────────────┘
```

**핵심 요소 명세**
- **중앙 뷰는 2모드 토글**: **성도(星圖) 뷰 = 기본** ↔ 카드 그리드 (분석·비교용). 토글 상태 기억.
- **성도 뷰 (C-08)** — 앱의 간판이자 공유 자산:
  - 레이더 동심원 3링(점수 90/70/50 등고선) + 회전 스윕. **점수가 높을수록 중심에 가깝게** 배치 — 원작 그래프와 달리 공간에 의미가 있다(레이더 은유: 강한 신호일수록 중심).
  - 키워드 노드 = 빛나는 별(글로우 필). 크기·밝기 ∝ 종합점수. 각도 = 클러스터(부채꼴 영역), 같은 클러스터는 성좌선(별자리 선)으로 연결.
  - 신규 발견 노드는 블립(깜빡임 3회) 후 정착. 스캔 중엔 스윕이 지나갈 때 노드가 점등.
  - 노드 클릭 → 우측 패널. hover → 이중 점수 미니 툴팁.
  - 60개 초과 시 외곽(저점수) 노드는 점으로 축약 + "더보기"는 카드 모드 유도.
- **TOP SIGNAL 스트립**: 오늘의 1위 키워드 + 한 줄 이유("AI 답변에 블로그 인용 0건 — 빈자리"). 마지막 스캔 시각, 다음 자동 스캔 카운트다운.
- **KeywordCard** (03 문서 C-01, 카드 모드): 키워드명(최대 2줄 말줄임) / DualScore 뱃지(N 87 · AI 92) / 7일 변동 스파크라인 / 클러스터 스트라이프. 클릭 → 우측 패널 로드. 카드 정렬: 종합점수 desc, 필터: 클러스터·AI빈자리만·급상승만 (필터는 두 모드 공통).
- **쿼터 게이지**: `오늘 스캔 186/999` 링. 소진 시 `지금 스캔` 버튼 disabled + 툴팁 "내일 0시에 충전돼요".
- **LIVE 티커**: 신규 발견만 표시(기존 재발견 제외). `prefers-reduced-motion` 시 정적 리스트로 대체.

**상태**
- 첫 스캔 중(온보딩 직후): 카드 그리드 자리에 레이더 스윕 애니메이션 + 진행 단계 텍스트("신호 수집 중 → 점수 계산 중 → AI 프로브 중"). 발견되는 대로 카드 실시간 추가 (스트리밍 UX — 3분을 지루하지 않게).
- 빈(스캔 후 0개): "이 주제는 신호가 약해요. 주제를 넓히거나 나눠보세요" + 추천 대안 주제 3개 버튼.
- 부분실패(AI 프로브만 실패): 카드에 네이버 점수만 표시 + AI 점수 자리 `측정 대기` 뱃지. **절대 0점으로 표시하지 않는다.**
- 에러(전체): 마지막 성공 스캔 결과 유지 + 상단 배너 "새 스캔 실패 — 이전 결과 표시 중 (재시도)".

**모바일 (기준 뷰포트 390×844 — 이 화면의 1차 설계 대상)**
```
[헤더 48px] 로고 · 주제칩 · 🔍 · [▶]
[TOP SIGNAL 콤팩트 1줄] 키워드 + N/AI 점수
[성도 뷰 — 정사각, 화면폭 100%]
   · 노드 탭 = 선택 (탭 타깃 최소 44px — 별 코어가 작아도 히트영역 확보)
   · 선택 시 하단 미니 요약바(키워드+이중점수) 슬라이드 업
   · 미니바 탭 → S3 풀 바텀시트(85%)
   · 핀치 줌·팬 지원 (외곽 저점수 노드 확인용), 더블탭 리셋
   · 라벨: TOP 3만 상시, 나머지는 선택 시
[수집 채널] 가로 스크롤 칩 4개 (신호등 점만)
[LIVE 티커 — 하단 고정 28px]
```
- 성도 모바일 성능: 노드 상한 40(초과분 카드 모드로), Pixi resolution `min(devicePixelRatio, 2)`, Bloom 품질 1단계 하향, 스윕 유지. 저사양 감지(프레임 드랍) 시 Bloom 자동 오프 — 화려함보다 60fps가 우선.
- 카드 모드: 1열 리스트. 성도↔카드 토글은 헤더 유지.
- hover 부재 대응: hover 전용 정보(sm 노드 라벨, 점수 근거 툴팁)는 전부 **탭 = 선택 표시**로 대체. hover에만 있는 정보 금지(공통 규칙에 추가).

## S2.5. 키워드 즉시 조회 (전역 검색 — REQ-C5)

**목적**: 이미 마음에 둔 키워드를 10초 안에 판정. 블랙키위 사용자가 넘어오는 첫 관문 — "쓰던 것보다 빠르고, AI 축이 하나 더 있네."

**진입**: 헤더 검색 아이콘 또는 **⌘K / Ctrl+K** (커맨드 팔레트 스타일 — 관제실 컨셉과 부합). 모든 화면에서 호출 가능한 오버레이 모달(max-w 640px).

**동작 (2단 제공 — 스캔 파이프라인과 동일 원칙)**
1. 키워드 입력 + Enter → **네이버 축 즉시** (~10초): DualScore의 N 점수 + 근거 3행(검색량/포화도/변동) 표시. AI 자리는 `AI 인용 가능성 확인` 버튼.
2. 버튼 클릭(옵트인 — 프로브 원가 통제) → 진행 표시(~2분) → AI 점수+프로브 결과 채움. 프로 플랜은 자동 실행.
3. 하단 액션: `주제로 등록`(이 키워드를 시드로 새 주제 — S1 스킵) / `브리핑 복사` / 최근 조회 5건 히스토리.

**상태**
- 조회 한도 소진: 결과 대신 플랜별 안내 "오늘 3회를 다 썼어요 — 스타터는 하루 20회" (업그레이드 유도, 화면 차단은 모달 내에서만).
- 자동완성 0개(니치): 시드 분해 폴백 대신 **단건 신호 수집으로 직행** (조회는 확장이 목적이 아님) + "연관어가 없는 키워드예요 — 수요가 매우 작을 수 있어요" 정직 안내.
- 실패: "네이버 응답이 늦어요" + 재시도. 이전 입력 유지.

**모바일**: 풀스크린 시트, 헤더 검색 아이콘으로 진입.

## S3. 키워드 상세 (우측 패널 / 모바일 바텀시트)

**목적**: "왜 이 점수인가"를 근거로 설득 → 글쓰기로 연결.

**구성 (위→아래)**
1. 키워드명 + DualScore 대형 표기 + 판정 라벨(점수 구간: 85+ `지금 쓰세요` / 70+ `좋은 기회` / 50+ `보통` / 미만 `관망`)
2. **근거 블록** (각 행: 지표명/값/한줄 해석)
   - 네이버 축: 월 검색량, 경쟁 포화도(문서수/검색량 → "검색 대비 글이 적어요"), 7일 변동
   - AI 축: 프로브 결과 요약 — "ChatGPT·퍼플렉시티에 물었을 때 블로그 인용 1건뿐" + 실제 질의문 표시(투명성)
3. **추천 앵글**: geo 룰 기반 1~2개 ("이 주제는 단계별 목록 형식이 인용에 유리")
4. 액션: `브리핑 복사`(마크다운 클립보드) / `이 키워드 추적`(프로: 발행 후 인용 추적 예약)

**상태**: AI 프로브 미완료 시 2번 AI 축 스켈레톤 + "측정 중 (~2분)". 프로브 실패 시 "이번 주 측정 실패 — 다음 스캔에 재시도" (재시도 버튼).

## S4. 글 진단

**목적**: 발행 전/후 글을 넣으면 AI 인용 가능성과 고칠 것을 준다.

**레이아웃**: 상단 URL 입력바(+ 최근 진단 히스토리 드롭다운) → 결과 2열: 좌 점수 링(대형 GaugeRing) + 인용 가능성 등급 / 우 개선 체크리스트.
- 체크리스트 항목: 엔진 aeo/geo analyzer 룰 매핑. 각 항목 = 상태 아이콘(통과/미흡/없음) + 항목명 + **"이렇게 고치세요" 구체 문장** + 예상 영향(높음/중간). boina의 정직성 원칙 계승: "고치면 인용됨" 단정 금지 → "인용 가능성을 높입니다".
- 진단 완료 시 `추적 목록에 추가` CTA (REQ-C3 연결 — 진단→추적 퍼널이 핵심 동선).

**상태**
- 진단 중: 단계 표시 "글 읽는 중 → 구조 분석 → AI 기준 대조" (60초 예상 바).
- URL 접근 불가: "글을 읽을 수 없어요. 공개 상태인지 확인해주세요" + 흔한 원인 2개(비공개/로봇 차단).
- 크롤 성공·분석 부분 실패: 성공한 항목만 표시 + 실패 항목 "측정 불가" 명시.

## S5. 인용 추적 (프로 핵심)

**목적**: "내 글이 AI에 인용됐다"는 순간을 만들어 보여준다 — 리텐션과 바이럴의 원천.

**레이아웃**
- 상단 요약 스트립: 추적 중 N개 / 이번 주 인용 M건 / 지난주 대비 화살표
- **타임라인 피드** (세로): 인용 이벤트 카드 = 날짜 / 어떤 AI(모델 뱃지) / 어떤 질문에 / 내 글이 어떻게 등장했는지(답변 발췌 하이라이트) / `자랑하기`(공유 카드 생성 — Should)
- 추적 대상 관리 테이블: 글 URL / 등록일 / 마지막 프로브 / 인용 누계 / 삭제

**상태**
- 인용 0건(대부분의 초기): 실망 방어가 관건 — "아직 인용 전이에요. 정상입니다: 새 글이 AI에 잡히는 데 보통 수 주 걸려요" + 지금 할 일(진단 점수 낮은 글부터 개선) 연결. **빈 화면을 코칭 화면으로.**
- 측정 방법론 각주 상시 노출: "매주 월 07:00, 질문 12종을 ChatGPT·퍼플렉시티에 질의해 확인합니다."

## S6. 주간 리포트

**목적**: 월요일 아침의 습관. 메일/알림 → 이 화면 랜딩.

- 주간 요약: 이번 주 키워드 TOP 5(이중 점수) / 내 인용 이벤트 / 놓친 기회(지난주 추천했는데 안 쓴 키워드의 이후 성과 — "쓸걸" 자극, 톤 주의: 죄책감이 아니라 다음 주 동기로)
- 과거 리포트 아카이브 리스트.
- Should: `이미지로 저장`(1080×1350 다크 카드) — 스레드 자랑용.

## S7. 설정 / 구독

- 주제 관리(추가/삭제 — 플랜 한도 표시), 알림 채널(메일 필수, 카카오 Should), 결제(현재 플랜/결제수단/해지).
- 해지 플로우: 1클릭 즉시 해지 + "다음 결제일까지 이용 가능". 다크패턴 금지(정직성 원칙) — 해지 사유 1문항(선택).
- 위험 상태: 결제 실패 시 앱 전역 배너(3일 유예) → 유예 후 무료 플랜 강등(데이터 보존).
- 탈퇴: 설정 최하단 — 해지 선행 확인 → 30일 유예 안내 → 확정. 약관·개인정보처리방침·환불정책 링크도 설정 하단 고정.

## 공통 규칙

- **모든 점수는 근거 접근 가능**: 점수 표시 요소는 클릭/hover 시 산출 근거 요약 툴팁.
- **카피 톤**: 관제실 컨셉이되 용어는 일상어("조준"·"신호" 같은 원작 군사 은유 금지 — 차별화 겸 이해도). 금지어: 인용 보장, 1위 보장, 무조건.
- **접근성**: 다크 배경 대비 텍스트 최소 4.5:1(03 토큰 준수), 모든 모션 reduced-motion 대응, 키보드로 카드 그리드 순회 가능(방향키+Enter). **성도 뷰의 접근성 대체 뷰 = 카드 모드** — 스크린리더·키보드 전용 사용자는 카드 모드를 기본으로 제공(성도는 캔버스라 보조기기 접근 불가, 동일 데이터의 시맨틱 뷰가 반드시 존재해야 함).
- **터치 규칙 (폰)**: ① 모든 정보·액션은 터치로 도달 가능 — hover는 **보조 강화용으로 허용**하되 hover에만 존재하는 정보는 금지 ② 탭 타깃 44×44px 이상 ③ 주 액션은 엄지 영역 ④ 바텀시트가 모달보다 우선.
- **데스크톱 강화 규칙 (PC)**: ① ⌘K 즉시 조회·방향키 카드 순회·`s`(스캔)/`d`(진단) 단축키 — 파워유저의 손이 마우스를 떠나지 않게 ② 1440·1920에서 여백만 늘어나는 게 아니라 **정보가 늘어난다**(성도 노드 라벨 상시 표시 수 확대, 카드 그리드 3→4열, 우측 패널 근거 전체 펼침) ③ hover 툴팁·프리뷰로 클릭 전 확인 ④ 브리핑 복사 후 "블로그 에디터 열기" 딥링크 — PC 작업 흐름의 마지막 연결 ⑤ 멀티 주제 탭 전환 (Ctrl+1~5).
- **성능 버짓**: 모바일(중급기) S2 LCP < 3s·성도 60fps(저사양 Bloom 자동 오프), 데스크톱 LCP < 2.5s·성도 Bloom 풀 퀄리티. 카드 가상화: 60개 초과 시.
