# 07. boina UX 재설계 — "진단 터널"에서 "우리 가게 홈"으로

> 대상: boina (소상공인판). 카피 원칙(사장님 언어·정직성·AI HERO 계층)은 계승하되,
> **[2026-07-04 개정] 비주얼·UI 레이어는 전면 교체** — `08-sme-design-system.md`("가게 성적표")가 디자인 정본.
> 인디고·이모지 UI·신호등 표현은 폐기, 3단 무공포 상태(●◐○)·증거 시트·변화 카드로 대체.
> 착수 시 boina `docs/planning/` Gap Review 패치의 원본 (06-tasks A0-1과 연결).
> 작성일: 2026-07-04

## 1. 무엇이 뒤죽박죽인가 (현행 구조 진단)

현행: `S1 찾기 → S2 내 상태 → S3 경쟁 → S4 갭 → S5 행동 → S6 생성물` 선형 위저드 + 라우트 7개(find/status/compare/gap/actions/checkout/settings).

| # | 문제 | 증상 |
|---|------|------|
| 1 | **6단계 터널** | 사장님이 답("뭘 해야 해?")을 보려면 화면 5개를 통과해야 함. 40~60대 사장님에게 단계가 많다 = 이탈 |
| 2 | **집이 없다** | 진단이 끝나면 랜딩할 홈이 없음. 재방문 시 "어디로 가지?" — 일회성 도구의 구조. 구독(레이더)을 붙일 자리도 없음 |
| 3 | **개념 과다 병렬** | 신호등 + AI HERO + 손실 프레이밍 + 갭 매트릭스 + 4분류 + 오늘 딱 하나 + 생성물 — 각각 좋지만 화면마다 새 개념이 등장해 "복잡한 보고서" 체감 |
| 4 | **상태와 행동의 분리** | S2(상태)와 S5(행동)가 3화면 거리. 사장님 머리엔 "그래서?"가 3번 쌓임 |
| 5 | **S3·S4 중복 서사** | "옆집이 낫다"(S3)와 "옆집은 갖췄다"(S4)는 같은 이야기의 두 장 — 나눠서 오히려 흐름 끊김 |

**한 줄 진단: 엔진의 분석 순서(크롤→점수→비교→갭→추천)대로 화면을 배열했다. 사장님의 질문 순서가 아니라.**

## 2. 재설계 원칙 — 사장님의 질문 순서로

사장님이 앱을 여는 실제 이유, 빈도 순:
1. **"오늘 뭐 하면 돼?"** (매번)
2. **"우리 가게 잘 보이고 있어?"** (매번, 확인 심리)
3. **"옆집은 어떤데?"** (가끔, 자극)
4. **"글감 줘"** (행동할 때)

→ 화면 구조도 정확히 이 순서. **행동이 1번, 상태가 2번, 경쟁이 3번.**

**AI HERO 원칙과의 조화 (기존 결정 존중)**: 기존 결정 "AI 노출이 HERO"는 유지하되, 홈의 HERO 카드는 상태 진술이 아니라 **상태+행동 융합 문장**으로 쓴다:
> "AI가 아직 우리 가게를 잘 몰라요. 👉 오늘 이거 하나 하면 알아보기 시작해요."
상태(정직)가 행동(오늘 딱 하나)의 이유가 되도록 — 개념 2개를 카드 1장으로.

## 3. 새 IA — 위저드 폐지, 홈 피드

```
[첫 방문 1회]                         [이후 모든 방문]
S1 가게 입력 → 분석 스토리(진행 연출) →  홈 (우리 가게)
                                       ├─ 상태 상세 (기존 S2 컴포넌트)
                                       ├─ 옆집 비교 (S3+S4 통합)
                                       ├─ 글감 만들기 (기존 S6)
                                       └─ 설정
하단 탭 3개: [우리 가게] [옆집] [글감]  (설정은 홈 우상단, 아이콘은 Material Symbols 사용)
```

라우트 7개 → **홈 + 상세 3 + 설정 = 5개.** checkout은 문맥 결제 시트로 흡수(별도 페이지 폐지).

### 홈 — "우리 가게" (단일 스크롤, 매주 갱신되는 집)

```
[헤더] ○○식당 · 6월 4주차                          [프로필]
┌─────────────────────────────────────┐
│ ① HERO — AI 상태 + 오늘 딱 하나 (융합 카드)        │
│ "AI가 아직 우리 가게를 잘 몰라요. 대부분 그래요.     │
│  👉 오늘은 이것만: 네이버플레이스에 메뉴 사진 3장"    │
│ [5분이면 돼요 — 바로가기]         5분 · 직접 가능   │
├─────────────────────────────────────┤
│ ② 우리 가게 채널 상태 (3줄 요약, 탭→상태 상세)      │
│ AI 추천     ○ 아직 준비 중이에요                   │
│ 네이버      ● 잘 보이고 있어요                     │
│ 구글        ◐ 조금만 더 채우면 돼요                 │
├─────────────────────────────────────┤
│ ③ 옆집 한 줄 (탭→옆집 비교)                       │
│ "△△식당은 챗GPT 추천에 나와요. 차이는 리뷰 답글."   │
├─────────────────────────────────────┤
│ ④ 이번 주 손님이 검색한 말 (레이더 모듈 자리)        │
│ ● ○○동 비오는날 맛집  [글감 만들기]                │
│ ● ○○동 혼밥          [글감 만들기]                │
│ (무료: 1개 미리보기 → "매주 5개 받기" 구독 CTA)     │
├─────────────────────────────────────┤
│ ⑤ 꾸준히 한 걸음 (리뷰 답글 미션 + 응원)             │
│ "이번 주 새 리뷰 2개 — 답글 달면 AI가 좋아해요"      │
└─────────────────────────────────────┘
```

- **④가 레이더 구독 모듈의 집이다.** 별도 S8/S9 화면 불필요 — 홈 카드가 매주 갱신되고, 알림톡은 홈으로 딥링크. (04 문서의 S8·S9는 이 카드의 상태 2종 — 미구독=제안 상태, 구독=리스트 상태 — 로 재정의)
- 홈의 카드 순서는 고정. "한 번에 하나" 원칙은 카드 단위로 유지 — 스크롤은 길어도 각 카드는 한 문장+한 행동.
- 매주 월요일 ①·④·⑤가 갱신 → **홈 자체가 주간 리포트.**

### 상세 화면 3종 (기존 자산 재배치 — 새로 만들지 않는다)

| 새 화면 | 기존 자산 | 변경점 |
|---------|----------|--------|
| 상태 상세 | S2의 AI HERO·채널 상태 데이터·실측 증거 | 정보 구조는 유지하되, 표현은 08의 도형 상태/증거 시트로 교체. 진입점만 홈 ②로 변경 |
| 옆집 비교 | **S3+S4 통합** | 한 화면 한 서사: 상단 "걔는 나와요, 당신은 아직"(S3 손실 카드) → 하단 "차이는 이것"(S4 갭 매트릭스를 '차이 목록 3개'로 단순화 — 매트릭스라는 형식 자체가 보고서 언어) → 맨 아래 "그럼 오늘 뭐?"(홈 ①로 회귀) |
| 글감 만들기 | S6 (스니펫 생성) | 진입점 다양화: 홈 ④ 검색어별 / 옆집 비교의 갭 항목별 / 글감 탭 직접 |

### 첫 방문 (S1 → 홈)

- S1(가게 입력 한 칸)은 유지. 위저드 제거의 핵심은 그 다음: 분석(수 분)을 **"단계 통과"가 아니라 "홈에 카드가 하나씩 도착하는"** 연출로. 채널 상태 도착 → 옆집 카드 도착 → 오늘 딱 하나 도착. 완성되면 알림.
- 분석이 끝나기 전에도 홈은 존재한다(스켈레톤 카드 + "찾는 중이에요") — 집부터 주고 가구를 채운다.

## 4. 상태 정의 (카드별 4상태 필수 — 크리에이터판과 동일 규율)

| 카드 | 로딩 | 빈/실패 |
|------|------|---------|
| ① HERO | "오늘 할 일 고르는 중" | 분석 실패 시: 어제 카드 유지 + "새로 확인 중이에요" |
| ② 채널 상태 | 상태 도형 회색 점멸 | 채널 측정 실패: 해당 줄만 "다시 확인할게요" (실패를 빨강으로 표시 금지 — 상태 의미 오염) |
| ③ 옆집 | 스켈레톤 1줄 | 경쟁사 미식별: "옆집 등록하면 비교해드려요" (입력 유도) |
| ④ 검색어 | 스켈레톤 2줄 | 신호 약한 주: "이번 주는 조용해요 — 리뷰 답글이 최고예요" (04 문서 규칙) |
