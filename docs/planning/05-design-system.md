# 05-Design System — 가게 성적표 UX

> 목적: 비IT 사장님이 점수·전문용어·공포형 경고 없이 `내 상태 / 라이벌 차이 / 오늘 할 일 / 이번 주 검색어`를 이해하고 실행하게 만드는 디자인 기준을 정의한다.

> 이 문서는 기존 원칙 1, 1-A, 2, 3의 방향은 유지하되, 시각 언어와 컴포넌트는 `docs/08-sme-design-system.md`의 `가게 성적표` 체계로 교체한다.

---

## 1. 양보 불가 원칙

| # | 원칙 | 의미 |
| --- | --- | --- |
| 1 | 사장님 언어 | SEO/AEO/GEO 같은 용어보다 `검색에 잘 보이나`, `옆집과 뭐가 다른가`, `오늘 뭘 하면 되나`로 말한다. |
| 1-A | AI와 검색은 결과가 아니라 재료까지 보여준다 | AI 노출만 과장하지 않고, 네이버/구글/SNS/리뷰/사진/메뉴가 AI와 검색에 잡히는 재료임을 설명한다. |
| 2 | 내가 직접 할 수 있나 | 모든 제안은 직접 할 수 있는지, 복붙하면 되는지, 꾸준히 쌓아야 하는지 구분한다. |
| 3 | 정직 | `이걸 고치면 1위`, `매출 상승 보장` 같은 인과 단정 금지. 수집/추정/실패/빈 결과를 구분한다. |

## 2. 시각 톤

| 항목 | 기준 |
| --- | --- |
| 제품 은유 | 가게 성적표. 무섭지 않고, 오늘 볼 과목과 다음 행동이 분명해야 한다. |
| 밀도 | 모바일 1열, desktop 중앙 560px column. desktop 여백에는 evidence/change panel만 둔다. |
| 카드 | 반복 항목과 sheet에만 사용. 섹션 전체를 카드 안에 다시 넣지 않는다. |
| 색 | deep green을 기준으로 하되 단색 테마처럼 보이지 않게 neutral, amber, blue, red를 제한적으로 쓴다. |
| 상태 | `● 좋음`, `◐ 손볼 곳`, `○ 준비 전` 3단계. 공포형 빨간 경고를 기본 상태로 쓰지 않는다. |
| 아이콘 | Material Symbols 또는 제품 icon set. 컨트롤 목적으로 emoji를 쓰지 않는다. |
| 숫자 | 숫자 점수는 primary UI에서 숨긴다. 필요한 경우 evidence sheet 안에서 보조 정보로만 둔다. |

## 3. 정보 계층

Home의 카드 순서가 제품의 정보 계층이다.

| 순서 | 계층 | 화면 | 역할 |
| --- | --- | --- | --- |
| ① | HERO + 오늘 하나 | home | 지금 가장 먼저 볼 변화와 오늘 할 한 가지 |
| ② | 채널 상태 | home/status | AI와 검색이 가게를 알아볼 재료 상태 |
| ③ | 라이벌 차이 | home/rivals | 경쟁자가 갖췄고 우리는 빠진 것 |
| ④ | 주간 검색어 레이더 | home | 이번 주 사람들이 찾는 말과 구독 상태 |
| ⑤ | 꾸준한 한 걸음 | home/write | 리뷰·사진·메뉴·소개글처럼 계속 쌓을 일 |

AI 노출은 중요한 결과지만, 단독 점수로 공포를 주지 않는다. `무엇을 채우면 AI와 검색이 가게를 더 잘 이해하는지`로 연결한다.

`docs/08-sme-design-system.md`의 M-01~M-07을 기준 컴포넌트로 사용한다.

| ID | 컴포넌트 | 용도 | 적용 화면 |

| M-01 | Status Banner | 지금 가장 중요한 상태와 오늘 하나 | home |

| M-07 | Reserved Sheet | 결제 제외 범위의 예약 슬롯. 현재 앱 표면에 노출하지 않음 | 없음 |

## 5. 상태 표현

| 상태 | 표시 | 의미 | 사용 예 |
| --- | --- | --- | --- |
| 좋음 | `● 좋음` | 지금은 크게 손볼 필요가 적다. | 네이버 기본 정보 충분 |
| 손볼 곳 | `◐ 손볼 곳` | 작은 수정으로 개선 가능성이 있다. | 사진 부족, 메뉴 설명 부족 |
| 준비 전 | `○ 준비 전` | 아직 재료가 없어 먼저 채워야 한다. | 리뷰 없음, SNS 연결 전 |

금지:

- `실패`, `망함`, `위험`을 기본 상태명으로 쓰지 않는다.
- 빨간색을 모든 부정 상태에 쓰지 않는다.
- emoji를 버튼, 탭, 상태 icon의 primary symbol로 쓰지 않는다.

## 6. 카피 기준

| 상황 | 좋은 카피 | 금지 카피 |
| --- | --- | --- |
| AI가 아직 모름 | `아직 AI가 가게를 잘 몰라요. 대부분 그래요. 지금부터 재료를 채우면 됩니다.` | `AI 노출 실패`, `이미 늦었어요` |
| 채널 부족 | `네이버 소개와 사진을 조금 더 채우면 검색이 가게를 이해하기 쉬워져요.` | `최적화 점수 38점` |
| 경쟁 열세 | `옆집은 메뉴 설명이 더 자세해요. 우리는 사진 설명부터 채워볼게요.` | `경쟁사 대비 -34% 열위` |
| 행동 권유 | `오늘은 사진 설명 하나만 고쳐요.` | `즉시 12개 항목 수정 필요` |
| 구독 안내 | `매주 월요일, 사람들이 이번 주에 찾는 말을 정리해드릴게요.` | `구독 안 하면 기회를 놓칩니다` |

## 7. 결제 제외 기준

2026-07-05 사용자 지시로 Toss 결제와 Kakao/SMS 알림은 현재 개발 범위에서 제외한다.

필수:

- 홈 카드 ④는 결제 CTA가 아니라 검색어 관심/확인 흐름으로 둔다.
- Settings는 결제수단, 구독 해지, 영수증, Kakao/SMS 수신 설정을 포함하지 않는다.
- `/checkout`은 직접 접근 호환용으로만 홈에 돌려보내며 primary IA에 노출하지 않는다.
- `/api/payment`는 주문 생성이나 권한 부여 없이 비활성 응답을 반환한다.

금지:

- Toss, Kakao/SMS, 알림톡, SMS fallback을 앱 표면에 노출하지 않는다.
- 정책 문구 없는 결제 CTA를 만들지 않는다.
- 결제 성공이나 구독 활성 상태를 표시하지 않는다.

## 8. Evidence 기준

M-04 Evidence Sheet에는 아래를 표시한다.

| 항목 | 설명 |
| --- | --- |
| 수집 시각 | 언제 확인했는지 |
| 출처 | 네이버/구글/SNS/AI 응답 등 |
| 원본 | 가능한 경우 URL 또는 질의 |
| 라벨 | 실측 / 수집 / 추정 / 실패 / 빈 결과 |
| 한계 | 수집 실패, 표본 부족, API 제한 등 |

근거 sheet는 공포를 줄이는 장치다. 사장님이 원할 때 펼쳐서 `왜 이렇게 말하는지` 확인할 수 있어야 한다.

## 9. 레이아웃 규칙

- 모바일: 한 화면 한 메시지, CTA는 엄지 영역에 둔다.
- Desktop: 중앙 560px column, 오른쪽에는 evidence/change panel만 둔다.
- 카드 radius는 8px 이하를 기본으로 한다.
- 버튼/탭/카드 텍스트는 줄바꿈을 허용하고 부모 영역을 넘지 않는다.
- 홈 카드 ④는 구독 CTA가 있어도 홈의 다른 카드보다 과도하게 커지지 않는다.

## 10. 정직성 가드

| 가드 | 규칙 |
| --- | --- |
| 인과 단정 금지 | `이거 고치면 1위/매출 상승` 금지 |
| 점수 primary 금지 | 숫자 점수는 내부 계산 또는 evidence 보조 정보로만 사용 |
| 실측 라벨 엄격 | 실제 수집하지 않은 값에 실측 표현 금지 |
| 빈 결과 정직 | 데이터가 없으면 없다고 말하고 다음 관찰 계획을 보여준다 |
| 구독 과장 금지 | 주간 검색어 구독을 매출 보장처럼 표현하지 않는다 |
