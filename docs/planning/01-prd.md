# 01-PRD — 소상공인 셀프서비스 검색·AI 가시성 진단 도구

> 목적: 소상공인이 "내 매장·브랜드가 검색·AI에 잘 보이는지 / 경쟁사보다 어떤지 / 그래서 뭘 해야 하는지"를 한눈에 확인하고 직접 행동하게 만드는 제품의 정본 요구사항을 정의한다.

> ⚠️ 제품명은 미결정([OPEN: 제품명] = OQ-1). 본 문서에서는 "본 제품"으로 표기한다.

---

## 1. 배경 — x-sag 반면교사

| 항목 | 내용 |
|------|------|
| 출발점 | `x-sag`(github.com/choiwjun/x-sag)의 분석 엔진은 유효하나, 제품 자체는 "이도저도 아닌 느낌"으로 한 가지 질문에 끝까지 답하지 못함 |
| 본 제품의 방향 | x-sag의 **정반대** — 단 하나의 질문("나는 검색·AI에 잘 보이나, 경쟁사보다 어떤가, 뭘 해야 하나")에 끝까지 답하는 집중형 제품 |
| 재사용 전략 | x-sag의 분석 엔진을 **부품으로 재사용**, 무거운 껍데기(external-API·화이트라벨·PDF·구독티어 복잡성)는 버리고 새로 짓는다 |
| 레포 | 별개의 새 git 레포 (x-sag와 분리) |

## 2. 문제

- 비IT 소상공인은 자기 매장이 네이버/구글 검색과 AI 답변에 **잘 노출되는지 알 방법이 없다**.
- 점수·전문용어 중심 도구는 사장님이 이해하지 못하고 "내가 직접 할 수 있나"라는 핵심 불안을 해소하지 못한다.
- 경쟁사와의 격차, 그 격차의 원인(경쟁사 셋팅), 그리고 "그래서 뭘 해야 하나"를 한 흐름으로 보여주는 도구가 없다.

## 3. 타깃 — 페르소나 P1

| 항목 | 내용 |
|------|------|
| P1 | 비IT 소상공인 (대부분 40~60대, 음식점·미용·카페 등) |
| 특성 | 마케팅 전문성 낮음, 주로 휴대폰으로 봄, 전문용어 모름, 바쁘고 자신 없음 |
| 핵심 불안 | "내가 직접 할 수 있나" |
| 판매 모델 | B2C 셀프서비스 직판 (대행사 화이트라벨은 **비범위**) |

## 4. 북극성 (North Star)

> 소상공인이 **내 매장·브랜드·제품이 검색·AI에 잘 보이는지 / 경쟁사보다 어떤지 / 그래서 뭘 해야 하는지**를 한눈에 본다.

|---|------|----------|

## 6. MVP 기능표 (REQ-001 ~ 007)

| REQ ID | 기능 | 설명 | 엔진 재사용 | MVP 상태 |
|--------|------|------|-------------|----------|
| REQ-001 | 가게 찾기/입력 | 가게 이름(+지역) 한 칸 → 네이버 플레이스 검색으로 식별(입력 최소). 홈페이지 URL은 선택 | naver-presence | 필수 |
| REQ-002 | 내 상태 진단 | SEO/AEO/GEO on-page 분석(엔진) + 네이버 노출 실측 + (조건부)구글 노출 + AI 실인용(grounded) → 신호등/한 줄 요약 | core-engine analyzers·scoring, llm-provider | 필수 |
| REQ-003 | 경쟁사 비교 | 경쟁사 노출 vs 나 (네이버 SERP·AI 추천에서 경쟁사가 위인가) — 손실 프레이밍 카드 | competitor·serp | 필수 |
| REQ-004 | 라이벌 역공학(갭) | 경쟁사를 같은 엔진으로 진단 → "경쟁사 보유 vs 내 갭" 매트릭스(GapAnalyzer) → "걔는 이걸 갖췄고 당신은 없음" | v2 GapAnalyzer | 필수(범위 [OPEN] = OQ-2) |
| REQ-005 | 행동 안내 | 모든 발견을 4분류(🟢🟡🔴⏳) + "오늘 딱 하나" 우선순위 + 직접 가능한 건 바로가기 딥링크 | — | 필수 |
| REQ-006 | 쉬운 것 생성 | 스니펫(FAQ/스키마)·플레이스 소개글·리뷰 요청 문구·업체 처방전(이메일 초안) 자동 생성 | snippet 로직 | 필수 |
| REQ-007 | 재진단/추이 | 개선 후 노출이 실제 올랐는지 재측정(구독 리텐션 hook 후보) | naver-presence 재호출 | **[OPEN] v1 포함 여부 = OQ-2** |

## 7. 성공지표

## 8. 스코프

### 채택 (In Scope)
- 셀프서비스 SEO/AEO/GEO 진단 (네이버 + 구글)
- 경쟁사 비교 + 역공학 갭 (x-sag GapAnalyzer 재사용)
- 4분류 행동 안내 + "오늘 딱 하나"
- 쉬운 생성물 (스니펫·소개글·리뷰 문구·처방전 이메일 초안)

### 기각 (Out of Scope — 스코프 규율)
| 기각 항목 | 사유 |
|-----------|------|
| 대행/매칭 마켓플레이스 | 제품이 무거워짐 |

| 매출/스마트플레이스 성과연결 | GEO 코어 이탈 + API 부재로 종결 |
