## Integration Rule (통합 규칙)

1. **구현완료 ≠ 코드 머지.** 각 REQ의 **구현완료 = 위 Hard/Metric/Domain/Evidence 게이트 증거 통과 전에는 불가**.
2. **upstream-gap 게이트**(OQ-3·OQ-4·OQ-6·P2-R4 배선·G-사장님이해·REQ-007)는 선행 결정/배선/검증 없이는 **풀 Green 불가** — 조건부 Green까지만 허용.
3. **G-HONESTY · G-사장님이해**는 개별 REQ Green과 **무관하게 항상 적용**되는 횡단 게이트다. 둘 중 하나라도 Red면 제품 전체가 출시 불가.
4. **v1.5 항목**(REQ-007, 구글 실 SERP/OQ-4)은 v1 구현완료 판정 대상에서 **제외**한다.
5. 본 게이트는 베이스라인 `06-tasks.md`의 V태스크/AC와 **충돌 시 더 엄격한 쪽**을 따른다 (충돌 우선순위: 사용자 결정 > 게이트 판정 > 베이스라인 > 08-derived-gates).
