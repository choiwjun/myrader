# 01. 크리에이터판 PRD — 서치레이더 (가칭)

## 1. 문제 정의

**처음 문제**: 크리에이터(블로거·스레드·유튜브 스크립트 작성자)는 "뭘 써야 조회수가 나오는지"를 감으로 정한다.
**재정의된 문제**: 검색 유입의 무게중심이 네이버에서 AI 검색(ChatGPT·퍼플렉시티·네이버 AI 브리핑)으로 이동 중인데, **크리에이터는 자기 글이 AI에 보이는지 확인할 방법 자체가 없다.** 기존 도구(블랙키위·키워드 레이더)는 전부 네이버 축만 본다.

**한 줄 정의**: "네이버와 AI 검색 양쪽에서 뜰 키워드를 찾아주고, 내 글이 실제로 AI에 인용되는지 추적해주는 웹 레이더."

## 2. 사용자 (JTBD)

**주 페르소나 — 수익형 블로거 "지현" (주 3~5회 발행)**
> 일요일 저녁 다음 주 글감을 정할 때(When), 조회수가 보장되는 키워드를 빨리 찾고 싶다(I want),
> 그래야 한정된 발행 횟수를 헛글에 낭비하지 않는다(So I can).
- 장면: 블랙키위 탭 6개 띄워놓고 엑셀에 검색량 복사 중. "이 짓을 매주 한다."
- 이미 유료 키워드 도구 결제 경험 있음. AI 검색 유입이 늘고 있다는 건 어렴풋이 앎.

**보조 페르소나 — 브랜드 운영 크리에이터 "민수" (자기 브랜드/뉴스레터)**
> 내 브랜드명을 ChatGPT에 물었을 때(When), 언급되는지 확인하고 언급되게 만들고 싶다(I want),
> 그래야 AI 검색 시대에도 유입이 끊기지 않는다(So I can).

## 3. 기능 범위

### Must (첫 출시 — 4개)

| ID | 기능 | 왜 필수 | 수용 기준 |
|----|------|---------|----------|
| REQ-C1 | **주제 레이더** — 주제 등록 → 키워드 발굴 + 이중 점수(네이버 기회/AI 인용 가능성) | 제품의 심장. 원작 대비 차별의 절반 | 주제 1개 등록 후 3분 내 키워드 15개+ 이중 점수 표시 |
| REQ-C2 | **글 진단** — URL 입력 → AI 인용 가능성 점수 + 개선 체크리스트 | 엔진(aeo/geo analyzer) 직결, 결제 이유 | 블로그 URL 진단 60초 내, 개선 항목 3개+ 제시 |
| REQ-C3 | **인용 추적** — 등록 글/브랜드가 AI 답변에 인용되는지 주간 측정 + 알림 | 해지 방어 장치이자 차별의 나머지 절반 | 주 1회 자동 프로브, 인용 발생 시 24h 내 알림 |
| REQ-C4 | **구독 결제** — 무료/스타터 9,900/프로 24,900 (토스 빌링) | 수익 | 카드 등록→익월 자동결제, 해지 즉시 반영 |
| REQ-C5 | **키워드 즉시 조회** — 아무 키워드나 입력 → 이중 점수+근거 즉시 확인 (주제 등록 불필요) | 크리에이터의 기존 습관("이 키워드 쓸까 말까")과의 브릿지. 블랙키위에서 넘어오는 첫 동선 | 입력 후 10초 내 네이버 점수+근거 표시, AI 점수는 옵트인 버튼(~2분) |

### Should (출시 후 1~2개월)
- 주간 리포트 공유 링크 (다크 카드 이미지 — 바이럴 자산)
- 키워드 → 추천 앵글("이 키워드는 비교표 형식이 인용 잘 됨") — geo 룰 기반
- 채널 연동 (블로그 RSS 자동 등록)

### Won't (명시적 OUT)
| 뺄 것 | 이유 |
|-------|------|
| 초안 생성 | 크리에이터는 자기 목소리로 씀. 소상공인판의 것 |
| 옵시디언 그래프 뷰 | 원작 시그니처 회피 + 개발비. 클러스터 그룹핑으로 대체 |
| 실시간 상시 스캔 | 쿼터·비용. 일 1회 자동 + 수동 스캔 (쿼터 표시) |
| 다국어/구글 SEO | 네이버+AI 축 집중. 확장은 지표 보고 |

## 4. 핵심 개념 정의 (구현 전 합의 필수)

**이중 점수**
- `네이버 기회 점수` (0~100): 검증 파이프라인 산식 — 연관강도·검색량(로그)·경쟁포화도(문서수/검색량)·트렌드 변동. `사장님레이더-검증/pipeline.js` 산식을 기준으로 튜닝.
- `AI 인용 가능성 점수` (0~100): geo-validator 프로브 결과 기반 — 해당 키워드 질의 시 ① 답변에 인용되는 소스 수(적을수록 빈자리) ② 인용 소스의 유형(공식문서 vs 블로그 — 블로그가 인용되는 주제면 기회) ③ aeo/geo 룰 충족 난이도. **"가능성"이라는 언어를 유지** — "인용 보장" 카피 금지 (정직성 원칙, boina 계승).

**인용(Citation)**: LLM 답변에 ① 내 도메인 URL 등장 ② 내 브랜드/블로그명 언급 ③ 내 글 고유 문구 등장 중 하나. 측정 방법론(질문 세트·주기·모델명)을 리포트에 항상 명시.

## 5. 수익·지표

| 플랜 | 가격 | 제한 |
|------|------|------|
| 무료 | 0 | 주제 1, 주 1회 레이더(키워드 5), **즉시 조회 3회/일**, 진단 월 2, 추적 없음 |
| 스타터 | 월 9,900 | 주제 2, 일 1회 레이더, **즉시 조회 20회/일**, 진단 월 20, 추적 글 3개 |
| 프로 | 월 24,900 | 주제 5, 일 1회+수동 스캔, **즉시 조회 무제한**(AI 프로브 포함), 진단 무제한, 추적 글 20개+인용 알림 |

**북극성 지표**: 주간 인용 발견 수 (사용자당 — "레이더가 성과를 증명한 횟수")
가드레일: 무료→유료 전환 5%+, 월 해지 10%↓, 진단 후 재방문율 40%+
