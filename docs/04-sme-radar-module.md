# 04. 소상공인판 — boina 레이더 구독 모듈

> boina에 추가되는 **주간 키워드 레이더 구독 기능**의 설계 정본.
> 착수 시 이 문서를 근거로 boina `docs/planning/`에 Gap Review 패치를 생성한다 (baseline patch-only 준수).
> 디자인: boina의 원칙(모바일 퍼스트·점수 숨김·정직성 카피)은 유지하되, 비주얼/UI는 `08-sme-design-system.md`의 "가게 성적표" 체계로 교체한다. 다크 관제실 아님.

## 1. 왜 이 모듈인가

boina 현 구조: 진단(S2)→경쟁 비교(S3·S4)→행동(S5·S6), **일회성 실행팩 결제**.
구조적 공백: 진단은 한 번 받으면 끝 — **재방문 이유가 없다.**
레이더 모듈 = "매주 우리 가게 뜰 검색어 + 복붙 글감"을 계속 배달하는 **구독 본체**. 진단(커피값)이 미끼, 레이더가 수익.

```
기존:  S1 입력 → S2 내 상태 → S3·S4 경쟁 → S5·S6 행동 → [끝]
추가:                                    S6 행동 ↘
                                          홈 카드 ④ 미구독 → 매주 홈 카드 ④ 구독 상태 → S6 재진입(복붙 생성물)
```
> ⚠️ 위 흐름도는 boina 구 IA 기준. **현행 구조는 07 문서(홈 피드)** — 레이더는 홈 카드 ④로 진입하며, 위 도표의 논리(진단=미끼, 레이더=구독 본체)만 유효하다.

## 2. 사용자 약속 (사장님 언어)

> "매주 월요일, 우리 가게 손님이 검색할 말과 바로 올릴 글감을 보내드려요."

- 점수·그래프·"키워드"라는 말도 최소화. **"손님이 검색하는 말"**로 부른다.
- 상태 표기: `● 지금 쓰면 좋아요` / `◐ 준비되면 쓰세요` / `○ 지켜보는 중` — 색만 쓰지 않고 도형+문장을 함께 쓴다.
- 정직성: "이거 쓰면 1위" 금지 → "요즘 검색이 늘고 글은 적은 말이에요"

## 3. 기능 명세

| ID | 기능 | 설명 | 수용 기준 |
|----|------|------|----------|
| REQ-R1 | 레이더 등록 | 진단 완료 가게는 **원클릭** (업종·지역·메뉴 이미 보유 — S1 입력 재사용). 시드 분해 확장 자동 적용 | 진단 이력 있으면 추가 입력 0개로 구독 시작 |
| REQ-R2 | 주간 스캔 잡 | 매주 월 06:00 KST 배치: 파이프라인 확장→점수→가게당 TOP5 선정→AI 인용 프로브(가게명)→리포트 생성 | 구독 가게 전체 완료 07:30 이전, 실패 가게 자동 재시도 1회 |
| REQ-R3 | 홈 카드 ④ 구독 상태 | 이번 주 검색어 5개(3단 상태) + 각각 "왜"(한 줄) + **복붙 글감 연결** | 알림 링크→3탭 이내에 복붙 완료 가능 |
| REQ-R4 | 복붙 글감 | 검색어 선택 → 기존 스니펫 생성기 재사용, 가게 정보 반영 블로그/인스타 글감 | 기존 snippets 모듈 호출, 신규 엔진 개발 없음 |
| REQ-R5 | 구독 결제 | 월 19,900 (단일 플랜 — 사장님에게 플랜 비교 강요 금지). 토스 **빌링(정기)** 추가 | 기존 일회성 결제와 병존, 해지 1클릭 |
| REQ-R6 | 알림 | 카카오 알림톡(1순위)·문자 폴백: "사장님, 이번 주 검색어 나왔어요" | 발송 성공률 95%+, 실패 시 문자 |

**Won't**: 사장님용 대시보드 상시 화면(주간 리포트만), 키워드 그래프, 다중 지점(프랜차이즈 — 지표 보고), 초안 무제한(글감 주 5개 고정 — 원가 통제).

## 4. 화면 (boina 화면 체계에 추가)

> **[2026-07-04 개정] `07-boina-sme-ux-redesign.md`의 홈 피드 구조가 확정되면서, 아래 S8·S9는 별도 화면이 아니라 홈 카드 ④("이번 주 손님이 검색한 말")의 두 가지 상태로 재정의된다:**
> - **S8(구독 제안) = 미구독 상태**: 실측 미리보기 1건 + 구독 CTA (홈 카드 안에서)
> - **S9(주간 리포트) = 구독 상태**: 검색어 5개 리스트 + 글감 연결 (홈 카드가 매주 갱신, 알림톡은 홈 딥링크)
> 아래 원문 명세는 카드 내부 구성·상태·카피의 정본으로 유지한다.

### S8. 레이더 구독 제안 (홈 카드 ④ 미구독 상태)
- 위치: S5/S6 "오늘 딱 하나" 완료 직후 — 행동 성공 경험 직후가 제안 최적점.
- 구성: 실제 미리보기 1개를 **공짜로 먼저** 보여준다 — "다음 주 사장님 가게엔 이런 말이 떠요: ● ○○동 혼밥" (파이프라인 실시간 1회 호출). 미리보기 아래 구독 CTA.
- 카피: "매주 월요일 아침, 이런 걸 5개씩 보내드려요 — 월 19,900원"
- 상태: 미리보기 실패 시 일반 예시로 폴백(가짜 데이터를 실측처럼 보이게 금지 — "예시" 라벨 필수).

### S9. 주간 레이더 리포트 (홈 카드 ④ 구독 상태)
```
[헤더] ○○식당 · 6월 4주차
[요약 문장] "이번 주엔 비 오는 날 검색이 늘었어요"
[검색어 카드 ×5]  ● ○○동 비오는날 맛집
                  "요즘 검색 늘고 글은 적어요"
                  [글감 만들기] ──→ 스니펫 화면(기존)
[지난주 돌아보기] 지난주 글감 쓰셨나요? (썼어요/아직요 — 원탭 응답, 리텐션 신호 수집)
[각주] 측정 기준 안내 (정직성)
```
- 상태 — 빈(신호 약한 주): "이번 주는 조용해요. 이럴 땐 단골 리뷰 답글이 최고예요" (대체 행동 제시, 빈손 금지) / 스캔 실패: 지난주 리포트 + "새 소식은 내일 다시 확인해드려요".

## 5. boina 통합 지점 (구현 가이드)

## 6. 원가·손익 (모듈 단독)

- 가게당 주간 원가: 파이프라인(네이버 API 무료 쿼터) + AI 프로브 ~12질의 + 글감 5건 LLM ≈ 100~200원/주
- 월 19,900 대비 원가 ~800원 → 마진 95%+. 무료 쿼터 한계: 데이터랩 1,000콜/일 → 스캔 설계상 **가게 ~600곳까지 무료 쿼터 내** (초과 시 유료 전환 계산 05 문서).

## 7. 지표

- 주간 리포트 열람률 60%+ / "썼어요" 응답률 25%+ (북극성: 사장님이 실제로 글을 올렸는가)
