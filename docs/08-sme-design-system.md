# 08. 소상공인판 디자인 시스템 — "가게 성적표"

> boina 기존 비주얼(인디고·이모지 UI)을 **대체**하는 전면 재설계. IA는 07 문서(홈 피드) 유지.
> 컨셉 한 줄: **은행 앱이 내 돈을 보여주듯, 이 앱은 내 가게가 보이는 정도를 보여준다.**
> 품질 기준점: 토스·배민사장님 — 40~60대가 매일 쓰는 유료 앱의 정제도.
> 실물: `design/boina-home-mockup.html` (v2 — 이 시스템으로 재제작)

## 1. 돈 받는 디자인의 5원칙

| # | 원칙 | 구현 |
|---|------|------|
| 1 | **3초 확인** | 첫 화면 최상단 = 이번 주 대표 상태 한 문장 ("이번 주 우리 가게, 잘 보이고 있어요"). 그 아래는 전부 근거와 할 일 |
| 2 | **증거 동봉** | 유료 사용자는 근거를 요구한다. 모든 판정 옆 `실제 확인 결과 보기` → AI에게 실제로 물어본 질문·답변 발췌 시트. "저희가 방금 물어봤어요"가 신뢰 = 결제 유지의 토대 |
| 3 | **변화의 가시화** | "지난주보다 ↑" 표기 + 월간 "한 일 → 바뀐 것" 요약. 돈 낸 보람이 화면에 보여야 해지하지 않는다 |
| 4 | **완료의 맛** | 오늘 할 일 = 체크 가능한 카드. 체크 순간 마이크로 피드백 + "AI가 알아보기까지 한 걸음" — 앱의 유일한 모션 |
| 5 | **장난감 금지** | 이모지를 UI 요소로 쓰지 않는다(콘텐츠 내 인용만 허용). 아이콘은 Material Symbols 단일 세트, 상태는 도형+색+문장 삼중 표기 |

## 2. 컬러 토큰

```css
:root{
  --bg:#F6F7F5;        /* 따뜻한 웜그레이 — 순백 배경보다 종이 성적표 느낌 */
  --card:#FFFFFF;
  --ink:#191F28;       /* 본문 잉크 (토스 계열) */
  --ink-2:#4E5968;     /* 보조 */
  --ink-3:#8B95A1;     /* 캡션 */
  --line:#E9ECEE;

  --brand:#0B7A55;     /* 딥 그린 — 성장·돈의 색. boina 인디고 대체 */
  --brand-deep:#075E42;
  --brand-soft:#E7F4EE;

  /* 상태 3단 — 빨강 없음 (사장님 앱에서 빨강 = 공포·이탈) */
  --good:#0B7A55;      /* ● 좋아요 */
  --mid:#C77700;       /* ◐ 보통이에요 (앰버 700 — 4.5:1 확보) */
  --wait:#6B7684;      /* ○ 준비 중이에요 (중립 — 실패가 아니라 시작 전) */
}
```
상태는 **도형+색+문장**을 항상 함께: `● 좋아요 / ◐ 보통이에요 / ○ 준비 중이에요`. 색맹·저시력 대응이자, 문장이 곧 사장님 언어.

## 3. 타이포그래피 (40~60대 기준 — 크게, 굵게, 짧게)

| 토큰 | 스펙 | 용도 |
|------|------|------|
| hero | Pretendard 800 · 22/32 | 대표 상태 문장 |
| title | 700 · 18/26 | 카드 제목 |
| body | 500 · **17**/26 | 본문 — 최소 17px, 15px 미만 금지 |
| caption | 500 · 14/20 | 근거·시각 표기 |
| number | 700 · tabular | "12% → 87%" 등 비교 숫자만 (점수 숨김 원칙 유지) |

문장 규칙: 한 카드 = 한 문장 + 한 행동. 조사·존댓말 완전체 ("~돼요" 체). 두 줄 넘는 설명은 시트로 밀어낸다.

## 4. 컴포넌트

### M-01 대표 상태 배너 (홈 최상단 — 3초 확인의 담당자)
흰 카드 아님 — **브랜드 그린 그라데이션** 배너. 내용 딱 3줄: 주차 라벨 / 대표 상태 문장(hero) / 지난주 대비 한 마디("지난주보다 한 칸 올랐어요 ↑"). 탭 → 상태 상세.

### M-02 할 일 카드 (오늘 딱 하나)
- 좌: 44px 체크 서클(탭 영역 56px). 우: 할 일 문장(title) + 이유 한 줄(caption) + `바로가기` 텍스트 버튼
- 체크 시: 서클 채움 애니메이션 400ms + 카드가 "완료" 상태로 접힘 + "내일 새 할 일이 와요"
- 난이도 라벨: `5분 · 직접 가능` (기존 4분류를 칩 하나로 압축 — 🟢🟡🔴⏳ 이모지 폐지)

### M-03 신호 행 (채널 상태)
`[아이콘 40px] 채널명(title) + 상태 문장(body) / 우측 ● 도형+미니 추세(↑→)` — 점수·게이지 없음. 탭 → 상세.

### M-04 증거 시트 (신뢰의 핵심)
바텀시트: 상단 "저희가 직접 물어봤어요" + 질문 원문(따옴표 블록) + AI 답변 발췌(내 가게/경쟁사 하이라이트) + 확인 시각. **꾸밈 없이 원문 그대로** — 이 정직함이 프리미엄이다.

### M-05 검색어 카드 (레이더 구독)
행: `● 검색어(title) / 이유 한 줄(caption) / [글감 만들기]`. 미구독 상태: 1행 실측 + 나머지 블러 처리 2행("매주 5개 받기" — 블러가 가장 정직한 티저, 가짜 예시보다 낫다).

### M-06 변화 카드 (월간 — 돈 낸 보람)
"이번 달 하신 일 4개 → 바뀐 것: 구글 ○→◐, 리뷰 답글률 12%→41%". Before→After 화살표 하나로. 그래프 금지(보고서 언어) — 문장과 도형 전환으로.

### M-07 결제 시트
가격 앞에 가치 재확인 1줄("매주 월요일, 손님이 검색하는 말 5개 + 글감") → 금액(number·크게) → 토스 결제 버튼 → "언제든 한 번에 해지돼요"(해지 쉬움을 **판매 문구로** — 신뢰가 전환을 올린다).

## 5. 모션·아이콘·여백

- 모션은 **체크 완료 1곳**만. 나머지 전부 정적 — 사장님 앱에서 움직임은 산만함이다.
- 아이콘: Material Symbols Outlined 단일 세트, 24px, `--ink-2`. 채널 아이콘만 브랜드 소프트 배경 40px 타일.
- 여백: 카드 패딩 20, 카드 간 12, 화면 좌우 20. 라운드 20(카드)·14(버튼)·풀(칩). 그림자 `0 1px 3px rgba(25,31,40,.06)` 한 단계만 — 떠 있는 느낌 금지, 종이 위 카드.

## 5.5 디바이스 정책 (크리에이터판과 의도적으로 다름)

소상공인판은 **모바일 우선을 유지**한다 — 사장님의 PC 접근은 드물고, 홈은 카톡 알림→폰 열람이 지배적 동선. 단 PC를 버리지는 않는다: 데스크톱(≥1024)에서는 홈 피드를 중앙 560px 칼럼으로 제시하고, 변화 카드·증거 시트를 옆 패널로 펼친다(월말 결산을 크게 보고 싶은 사장님·직원과 함께 보는 상황). 전용 데스크톱 기능은 만들지 않는다.

## 6. 기존 boina 디자인에서 유지 vs 폐기

| 유지 (원칙) | 폐기 (표현) |
|------------|------------|
| 사장님 언어·전문용어 0 | 인디고 프라이머리 → 딥 그린 |
| 정직성 카피 가드 전체 | 이모지 UI (🟢🟡🔴⏳·표정) → 도형+문장+아이콘 |
| AI HERO 정보 계층 | 신호등 은유의 "빨강" → 3단 무공포 상태(good/mid/wait) |
| 점수 숨김 | 화면당 개념 병렬 → 카드당 한 문장 |
| 모바일 퍼스트·큰 글씨 | Plus Jakarta Sans 혼용 → Pretendard 단일 (숫자만 tabular) |

> boina Gap Review 패치 시 `05-design-system.md`의 원칙 절(1·1-A·2·3)은 보존하고 비주얼 절만 이 문서로 교체한다.
