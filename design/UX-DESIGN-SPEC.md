# 보이나 — UI/UX 후킹 설계 명세 (v1)

> **정직 후킹 × 반응형 × 고급 퀄리티.** 화면마다 "심리적 임무"를 박는다.
> 이 문서는 디자인 도구(pencil / Stitch / 코드)에 넘기는 **단일 정본 브리프**다.
> 도구는 룩 탐색용이고, 최종은 이 스펙을 강제한 반응형 코드다.

---

## 0. 절대 전제 (바뀌면 제품 정체성이 무너짐)

1. **정직성 가드(AC-8)** — 인과 단정("고치면 1위")·가짜 숫자·가짜 후기·**가짜 합의("너만 늦었다")** 0. 후킹은 **화이트햇만**.
2. **전문용어 0** — GEO/AEO/SEO/스니펫을 사용자에게 **노출 안 함**. 용어를 가르치지 말고 **느끼게** 한다.
3. **익명 무료 진단 = 가치 먼저** — 가입·결제 전에 가치를 다 보여준다(안티 다크패턴). 이게 신뢰 후킹의 뼈대.
4. **반응형 1급** — 모바일·데스크톱 둘 다 1급 시민. 데스크톱에서 430px 회색 프레임은 **폐기**.
5. **40~60대 비IT 사장님** — 큰 글씨·큰 터치·응원 톤. 회의적 페르소나(대행에 데인) 전제.

## 1. 후킹 도구상자 (화이트햇 7 + 금지 다크패턴)

### 쓸 것 (정직 + 회의 페르소나 OK)
| 기술 | 정의 | 우리 적용 |
|------|------|-----------|
| **호기심 갭(open loop)** | 미완의 정보로 닫고 싶게 | "내 가게가 AI에 어떻게 보이는지 1분이면 봐요" |
| **손실 프레이밍(실데이터)** | 추상 공포 ✕ / 구체 손실 ○ | "실제 옆집 OO카페는 뜨는데 당신은 아직" |
| **진행감(goal gradient)** | 끝이 보이면 끝까지 감 | 단계 표시 · "다 봤어요" |
| **작은 약속(commitment)** | 작은 행동 → 몰입 | 가게 이름 한 칸 (마찰 최소) |
| **절정-결말(peak-end)** | 마지막 인상이 기억을 지배 | "오늘 딱 하나" = 작은 승리로 끝맺음 |
| **선점 프레임(정직한 사회적 증거)** | 거짓 합의 ✕ / 진짜 기회 ○ | "아직 아무도 안 해요 = 지금이 기회" |
| **노동/소유 효과(endowment)** | 내가 만든 것에 애착 | 내 가게 데이터·복붙 자산 = "내 것" |

### 절대 금지 (쓰는 순간 "그들이 싫어하는 그 업체"가 됨)
가짜 카운트다운 · 가짜 사용자 수 · **"너만 늦었다"(거짓)** · confirmshaming · 가치 보기 전 강제가입 · 다크 구독 함정.

## 2. 전환 퍼널 — 감정 아크

```
랜딩 ──► 찾기 ──► 진단중 ──► 내상태 ──► 경쟁비교 ──► 갭 ──► (페이월) ──► 행동 ──► 생성물 ──► 재방문*
호기심   작은약속   기대      안심·각성   손실=각성    욕구    전환        작은승리   효능감     애착
낮은마찰                     (절정 비주얼)            (호기심갭)
```

감정 곡선: **호기심↑ → 안심(당신 탓 아님) → 가벼운 각성(실 손실) → 욕구(더 보고 싶다) → 해소(오늘 딱 하나) → 효능감(복붙) → 애착(내 데이터)**

핵심 설계 룰:
- **불안은 추상이 아니라 실데이터에서만** 발생시킨다(경쟁비교). 차가운 랜딩에서 겁주지 않는다.
- 각 화면은 **다음 화면을 여는 open loop** 를 하나씩 남긴다.
- **"오늘 딱 하나"가 감정의 종착점** — 무력감이 아니라 "이거 하나면 된다"는 통제감으로 끝낸다.

## 3. 화면별 정밀 설계

> 형식: **심리 임무 / 1차 후킹+배치 / 정보 위계 / 핵심 카피 방향 / 상태(빈·로딩·에러) / 반응형 거동 / 다음 연결**

### 랜딩 (`/`)
- **심리 임무**: 호기심 + 낮은 마찰 → "1분인데 한번 볼까?"
- **후킹**: 헤드라인 = 호기심 갭("AI한테 물어볼 때 내 가게는?") · CTA = 무료·1분·이름 한 칸 · 작게 선점 프레임("대부분 아직 안 봤어요, 지금 보는 게 빠른 거예요")
- **위계**: ① 헤드라인(스테이크) ② 큰 CTA ③ 4스텝 미리보기 ④ 안심줄 ⑤ 베타 정직 표기
- **카피 방향**: "요즘 손님은 검색하고, AI한테 물어보고 가게를 골라요. 거기서 내 가게가 어떻게 보이는지 — 1분이면 봐요. 무료."
- **상태**: 정적(상태 없음)
- **반응형**: 모바일=단일 세로 / 데스크톱=중앙 히어로 + 우측 시각(목업·신호등 미리보기) 2단. **회색 430px 프레임 폐기.**
- **다음**: CTA → `/find`

### S1 가게 찾기 (`/find`)
- **심리 임무**: 작은 약속(commitment) — 한 칸 입력으로 몰입 시작
- **후킹**: 입력 필드 1개(이름) + 지역/URL 선택 · 진행감(검색→후보→확정→진단중 4단계 가시화)
- **위계**: ① 단일 입력 ② 후보 카드 ③ 시작 버튼
- **카피**: "가게 이름만 있으면 돼요." / URL "없어도 돼요"
- **상태**: 빈(후보 0 "못 찾았어요, 이렇게 해볼까요?") · 로딩(찾는 중) · 에러(친절 재시도)
- **반응형**: 모바일=세로 스텝 / 데스크톱=입력 좌측 + 후보 우측 리스트
- **다음**: 진단 완주 → `/status` (open loop: "다 봤어요")

### S2 내 상태 (`/status`)
- **심리 임무**: 검증 + 안심 + **절정 비주얼**(브랜드 임팩트가 가장 큰 화면)
- **후킹**: AI HERO(시그니처 비주얼) · "당신 탓 아님" 검증("아직 대부분 가게가 그래요") · 선점("지금 준비하는 가게가 먼저 잡혀요")
- **위계**: ① AI HERO(상태 한 문장+신호등) ② 채널(네이버/구글/AI)=연료 ③ 다음 CTA
- **카피**: "AI가 우리 가게를 조금씩 익히는 중이에요" + 정직("실제로 AI가 추천할 때만 초록불")
- **상태**: 로딩(스켈레톤 HERO) · 에러(가게정보 없음→/find)
- **반응형**: 모바일=HERO 풀폭 / 데스크톱=HERO 좌측 대형 + 채널 카드 우측 정렬
- **다음**: "옆집과 비교해 볼까요? 👀" → `/compare` (open loop)

### S3 경쟁 비교 (`/compare`) — ★ 진짜 후킹 모먼트
- **심리 임무**: **손실 프레이밍으로 각성** — "어 이거 진짜 문제네"
- **후킹**: 실 옆집 vs 우리(신호등) · 손실 헤드라인("옆집은 네이버에서 먼저 보이고 있어요") · 출처 배지(정직)
- **위계**: ① 손실 헤드라인 ② 옆집⭕/우리✕ 카드 ③ 출처 ④ 다음 CTA
- **카피**: "(샘플)" 라벨 정직 · 인과 단정 0
- **상태**: 빈(경쟁사 없음 "잘 지키고 계세요!") · 로딩 · 에러
- **반응형**: 모바일=세로 카드 / 데스크톱=옆집|우리 좌우 대비 강조(시각적 격차가 핵심)
- **다음**: "옆집은 뭘 갖췄나 볼까요? 🔍" → `/gap` (강한 open loop)

### S4 역공학 갭 (`/gap`)
- **심리 임무**: 욕구 증폭 + 호기심 갭 → 전환(페이월)
- **후킹**: 무료 Top3로 가치 증명 → 나머지 잠금(더 보고 싶음) · 정직 면책("따라한다고 바로 효과 X")
- **위계**: ① 갭 요약 ② 무료 Top3 매트릭스 ③ 페이월(손실 프레이밍, 가격 미표시) ④ 다음 CTA
- **카피**: 룰코드값 0, 사장님 언어
- **상태**: 빈(갭 0 "잘 갖추고 계세요") · 로딩 · 에러
- **반응형**: 모바일=세로 / 데스크톱=매트릭스 표 형태로 한눈에 + 잠금 영역 블러
- **다음**: 갭 카드 → `/actions?tier=` (해당 4분류 섹션) · 잠금 → `/checkout`

### S5 행동 (`/actions`) — peak-end
- **심리 임무**: **작은 승리로 끝맺음** — 무력감 ✕, 통제감 ○
- **후킹**: "오늘 딱 하나"(von Restorff 단일 강조) · 4분류(🟢직접 바로가기/🟡복붙/🔴업체/⏳꾸준히)
- **위계**: ① 오늘 딱 하나 배너 ② 🟢직접(딥링크) ③ 나머지 분류 ④ 페이월
- **카피**: "오늘은 딱 하나만 해봐요" · 보장 단정 0
- **상태**: 빈(행동 0 "아직 추천할 행동을 못 찾았어요") · 로딩 · 에러
- **반응형**: 모바일=세로 섹션 / 데스크톱=오늘딱하나 상단 풀폭 + 4분류 2열 그리드
- **다음**: 🟡🔴 → `/assets?type=` (즉시 효능감)

### S6 생성물 (`/assets`)
- **심리 임무**: 즉시 효능감 + 소유 효과 → "이거 바로 쓸 수 있네"
- **후킹**: 복붙 가능 4종(소개글·리뷰요청·스니펫·업체처방전) · 큰 복사 버튼 · "그대로 쓰고 다듬으면 더 좋아요"
- **위계**: ① 사용법 한 줄 ② 자산 카드(복사) ③ 페이월
- **상태**: 빈 · 로딩 · 에러
- **반응형**: 모바일=세로 카드 / 데스크톱=카드 2열 + 미리보기 넓게
- **다음**: 종착(이전으로 자유 왕복) · 재방문 hook(추이, v1.5)

### S7 설정 (`/settings`)
- **심리 임무**: 통제감·소유 (보조 화면)
- **후킹**: 내 가게 정보 관리(업종 포함) · 재진단 placeholder("곧")
- **반응형**: 모바일=세로 폼 / 데스크톱=2열 폼
- 주: 로그인 필요(계정 전략 결정 후 진입 경로 확정)

### 공통 — 헤더 / footer / 로그인 / 약관
- 헤더: 브랜드 + 가게명(이메일 ✕) + 설정. 데스크톱=상단 네비 확장 여지.
- footer: 운영주체(오픈 전 실값) + 약관/개인정보.
- 로그인/약관/개인정보: 신뢰 골격(이미 구현, 비주얼 격상 대상).

## 4. 디자인 시스템 골격 (도구가 만들 프리미티브)

> 색/모션의 **구체 값**은 §6 "디자인 무드" 확정 후 채운다. 여기선 **역할·구조**만 고정.

- **컬러 역할**: 신호등(🟢🟡🔴⏳ — 의미 고정) · 브랜드(1 primary) · 중립(텍스트/배경 스케일) · 강조(CTA) · 표면(카드/깊이)
- **타이포 스케일**: 디스플레이(HERO) / H1·H2 / 본문(16px+) / 캡션. 한글 가독 우선(큰 글씨).
- **간격 리듬**: 4/8px 베이스. 카드 패딩·섹션 간격 일관.
- **깊이/그림자**: 플랫 탈피 — 미묘한 elevation으로 위계.
- **모션 원칙**: 절제된 트랜지션(화면 전이·복사 피드백·신호등 점등). 과한 애니메이션 ✕.
- **아이콘**: **이모지 → 실 아이콘 세트**(lucide 등 일관 라인). 신호등만 의미상 컬러 도트 유지.
- **컴포넌트 인벤토리**: AppHeader · AI HERO · SignalLight · 비교카드(옆집vs나) · 갭매트릭스 · ActionCard(4분류) · TodayOneBanner · BigCopyButton · PaywallGate · StepNav · SiteFooter · 입력필드 · 후보카드 · 스켈레톤/빈/에러 상태.
- **반응형**: 브레이크포인트(모바일 / 태블릿 / 데스크톱). 모바일=단일 컬럼, 데스크톱=콘텐츠 max-width(예 1080~1200) + 다단/사이드 레이아웃. **430px 회색 프레임 제거.**

## 5. 도구 브리프 (pencil / Stitch / 코드 공통)

- 이 문서가 **단일 정본**. 도구는 §3 화면별 "위계+후킹+카피 방향"을 그대로 프롬프트/생성 입력으로 쓴다.
- 순서: ① §6 무드 확정 → ② 디자인 시스템(색/타이포/모션) 정의 → ③ **랜딩 + 경쟁비교(후킹 핵심 2화면)** 고화질 반응형 먼저 → ④ 합격 시 전 화면 롤아웃.
- 합격 기준: (a) 후킹 장치가 §3대로 배치됐나 (b) 정직성 가드 위반 0 (c) 모바일·데스크톱 둘 다 1급 (d) "진짜 제품" 퀄리티(이모지 0·플레이스홀더 0·일관 시스템).
